## File naming

Libraries that interact with a product often name their packages after that product. If you name a Python file `<PRODUCT_NAME>.py` and then try to import the library of the same name from within it, you will get import errors that are difficult to diagnose.

<Warning>
Never name a Python file the same as the integration's name.
</Warning>

For example, if you are writing an integration for a product called `awesome`, do not create a file named `awesome.py` inside the integration package. Use a more specific name such as `check.py` or `client.py` instead.

## Attribute naming

The base `AgentCheck` classes may freely add new attributes for new features at any time. To avoid collisions with future additions, prefix your attribute names with underscores — especially for generic names.

```python
# Prefixed to avoid collision with any future base class attribute
self._server = self.instance.get('server', '')
self._port = int(self.instance.get('port', 8080))
```

## Stateful checks

Since Agent v6, every instance of `AgentCheck` corresponds to a single YAML instance of an integration defined in the `instances` array of user configuration. Because each instance object persists for the lifetime of the Agent, the `instance` argument passed to the `check` method is redundant — you are parsing the same configuration on every run.

**Parse configuration once in `__init__` and save the results as instance attributes.**

<Tabs>
  <Tab title="Do this">
    ```python
    class AwesomeCheck(AgentCheck):
        def __init__(self, name, init_config, instances):
            super(AwesomeCheck, self).__init__(name, init_config, instances)

            self._server = self.instance.get('server', '')
            self._port = int(self.instance.get('port', 8080))

            self._tags = list(self.instance.get('tags', []))
            self._tags.append('server:{}'.format(self._server))
            self._tags.append('port:{}'.format(self._port))

        def check(self, _):
            ...
    ```
  </Tab>
  <Tab title="Do NOT do this">
    ```python
    class AwesomeCheck(AgentCheck):
        def check(self, instance):
            server = instance.get('server', '')
            port = int(instance.get('port', 8080))

            tags = list(instance.get('tags', []))
            tags.append('server:{}'.format(server))
            tags.append('port:{}'.format(port))
            ...
    ```
  </Tab>
</Tabs>

<Note>
Notice the underscore prefixes on `self._server`, `self._port`, and `self._tags`. This follows the [attribute naming convention](#attribute-naming) and prevents collisions with base class attributes that may be added in the future.
</Note>

## Check initializations

When setup work needs to be deferred — for example, creating a connection that requires the Agent to be fully started — use the `check_initializations` list provided by the base class rather than putting setup logic directly in `__init__`.

Add callables to `self.check_initializations` inside `__init__`. The base class calls each one before the first `check()` run and will retry them on subsequent runs if they raised an exception.

```python
class AwesomeCheck(AgentCheck):
    def __init__(self, name, init_config, instances):
        super(AwesomeCheck, self).__init__(name, init_config, instances)

        self._server = self.instance.get('server', '')
        self._client = None

        self.check_initializations.append(self._connect)

    def _connect(self) -> None:
        self._client = create_client(self._server)

    def check(self, _) -> None:
        # self._client is guaranteed to be initialized here
        ...
```

## Configuration models

<Warning>
Do not modify files in `**/config_models/*.py` directly. Those files are generated from `assets/configuration/spec.yaml`.
</Warning>

To change a configuration model, edit `assets/configuration/spec.yaml` and then regenerate the models:

```bash
ddev -x validate config -s <INTEGRATION_NAME>
ddev -x validate models -s <INTEGRATION_NAME>
```
